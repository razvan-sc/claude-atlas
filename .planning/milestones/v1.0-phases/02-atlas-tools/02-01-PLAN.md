  <behavior>
    - get_project("proj-123") returns JSON with name, description, contributors (list of names), and target date
    - get_project("nonexistent") returns an error string starting with "Error:"
    - get_projects(["proj-1", "proj-2"]) returns JSON array with both projects
    - GraphQL response data is extracted and flattened into readable output (not raw nested GraphQL structure)
  </behavior>
  <action>
    1. Create `src/atlas/queries.py` with GraphQL query strings as module-level constants:
       - `GET_PROJECT_QUERY`: Uses `projects_byId` operation. Request fields: `key`, `name`, `description`, `state { value }`, `targetDate`, `contributors { edges { node { name, aaid } } }`. The Atlas GraphQL API nests project data under `data.project.projects_byId`.
       - `GET_PROJECTS_QUERY`: Uses `projects_byIds` operation with `$ids: [ID!]!`. Returns same fields as above for each project. Data nested under `data.project.projects_byIds`.

    2. Add two new MCP tools in `src/atlas/server.py` (keep existing `atlas_graphql_query` tool):
       - `get_project(project_id: str) -> str`: Accepts a single project ID. Calls `client.execute(GET_PROJECT_QUERY, {"id": project_id})`. Extracts from `result["data"]["project"]["projects_byId"]`, formats as JSON with keys: name, description, state, targetDate, contributors (flattened to list of names). Returns `json.dumps(formatted, indent=2)`. Catches exceptions with the established `f"Error: {type(e).__name__}: {e}"` pattern.
       - `get_projects(project_ids: list[str]) -> str`: Accepts list of project IDs. Calls `client.execute(GET_PROJECTS_QUERY, {"ids": project_ids})`. Extracts from `result["data"]["project"]["projects_byIds"]` (list). Formats each project the same way. Returns `json.dumps(formatted_list, indent=2)`.
       - Both tools: use `load_config()` then `async with AtlasGraphQLClient(config) as client:` (same pattern as existing tool).

    3. Create `tests/test_project_tools.py`:
       - Use the same `httpx.MockTransport` testing pattern from `test_graphql_client.py`.
       - Instead of testing MCP tools directly, test helper functions. Extract the GraphQL call + response formatting into importable async functions in `server.py` (or keep tools thin and test the formatting logic). Preferred approach: the MCP tool functions ARE the testable functions -- import them directly and call with await. They use `load_config()` internally, so mock the config file using `tmp_path` fixture and monkeypatch.
       - Alternative simpler approach: create thin internal async functions `_get_project_impl(client, project_id)` and `_get_projects_impl(client, project_ids)` that take an already-constructed client. Tools call these. Tests call these directly with a mock-transport client.
       - Write tests for: successful single project query, successful multi-project query, error handling (GraphQL error returns error string).
